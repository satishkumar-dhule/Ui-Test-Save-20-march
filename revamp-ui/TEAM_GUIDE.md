# UI/UX Revamp — Squad Guide (Autonomous Delegation)

Overview

- 6 squads (S1–S6), each 5 people: Lead, 2 Frontend Engineers, UX Designer, QA/Automation
- Central governance: Design Systems Studio, UX Research Pod, Theme/Accessibility Champion
- All squads report to Head of UI/UX Revamp; DS Studio and UX ResearchPod operate cross-squad

Squad structure (per squad)

- Squad Lead (PM/Tech) – owns scope, milestones, risk, tech decisions
- FE1, FE2 – implement components and flows
- UX Designer – user research, flows, wireframes, interaction design
- QA/Automation – tests, accessibility checks, regression automation

Cadence

- 2-week sprints; Sprint rituals on fixed calendar
- Weekly squad planning & demos
- Biweekly cross-squad design reviews
- Biweekly DS Council (tokens, migrations)
- Monthly UX Research sprint

Initial responsibilities (Phase 0)

- S1–S6: Build token-driven UI kit and core components; onboarding prototype
- DS Studio: establish token schema, component contracts, docs
- UX Research Pod: recruit participants, run first usability test on onboarding
- Theme/Accessibility Champion: define baseline accessibility standards

Delivery expectations

- A living design system with tokens, components, and usage docs
- Reusable onboarding flow and at least one additional canonical flow demonstrated in a tiny prototype
- CI: UI tests, accessibility checks, and visual diff checks for new components

Artifacts to reference

- revamp-ui/design-tokens.json and revamp-ui/styles/global.css
- Core components at revamp-ui/src/
- Onboarding flow at revamp-ui/flows/Onboarding.tsx
- DS governance artifacts and migration plan (to be populated)

Notes

- This guide assumes an autonomous rollout; team leads will drive day-to-day without further confirmations.
- We will adjust as needed based on early results and pilot feedback.

Acceptance Criteria (sample)

- Tokens expanded: typography scales, elevation tokens; dark mode tokens present
- Modal, Dropdown, Avatar, Tag components render with tokens and accessible props
- Shell provides header, nav, and content area; responsive and accessible
- Flow UIs render in a preview shell with token-driven styling
- Storybook shows all new components and flows; builds cleanly
- Handoff docs updated; contracts defined; visual-diff hooks in place
- Accessibility checks cover keyboard navigation and aria attributes
- Phase 1 demo showcases two flows in a cohesive shell

Definition of Done (DoD)

- Phase 1 tokens finalized and committed; components implemented and wired to tokens
- At least 3 canonical flows implemented as preview pages
- Storybook stories for new components and flows present
- Handoff/docs updated; migration plan drafted
- Accessibility baseline satisfied; keyboard and ARIA supported
- Phase 1 demo script prepared for leadership review

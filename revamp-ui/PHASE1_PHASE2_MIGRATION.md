# Phase 1 to Phase 2 Migration Plan

Objective

- Define concrete steps to migrate from Phase 1 token-based UI to Phase 2 preview and expanded flows, with minimal risk.

Scope

- Extend flows to 4–5 canonical flows; add a lightweight analytics hook; enable more components; prepare ongoing migration path.

Key steps

- Phase 2 token expansion: additional tokens for new components
- Phase 2 components: Card, Avatar, Tag already present; add Modal variants, Dropdown variants, Drawer, Tabs
- Phase 2 flows: add 1–2 new flows (e.g., Notifications center, Settings revamp)
- Migration plan: snapshot of current Phase 1 usage, plan steps to Phase 2, with rollback.
- Governance updates: update DS Council artifacts, migration docs, and DOD gates.

Risks and mitigations

- Token complexity: incremental tokens; avoid explosion; keep a single source of truth
- Migration friction: provide migration path with clear deprecations and migration UI
